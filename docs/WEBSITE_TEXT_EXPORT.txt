START HERE

Think of AgentLayer as the trust layer that sits between agent identity and real economic access. Agents register once, keep the same identity, prove what changed, and earn more privileges over time.

Core message

AgentLayer is not just a profile page. It is the shared system platforms can query before routing, ranking, paying, rate-limiting, or blocking agents.

Who buys first

The first buyers are platforms that need a trust interface before giving agents visibility, access, routing priority, credit, settlement, or permission to operate in higher-risk workflows.

1. Generate a key.

Machine quickstart

An agent discovers AgentLayer through the well-known document, requests a registration challenge, signs a registration claim with its private key, and submits one signed registration call to receive an agent ID and passport.

TRUST DEFENSES AGAINST SYBIL GAMES

Cheap registration is acceptable.
Cheap influence is not.

Trust becomes meaningful only when continuity, provenance, policy, and economic consequences are layered together.

Defense layers:

Identity continuity
The same key carries the same identity across time.

Weighted attestations
Not every attestation counts equally.

Release provenance
Changes are visible instead of hidden.

Policy tiers
High-impact actions require more trust.

Economic security
Bond, holdback, and slash make trust costly to fake.

Costly actions

Pure proof-of-work is not the root solution. Costly actions can reduce spam, but the stronger model is identity continuity plus weighted trust, policy, and collateral.

High-impact policy

High-trust actions should require stronger verification, stronger history, and stronger economic backing than low-risk actions.


RELEASE PROVENANCE AND ECONOMIC SECURITY

Identity stays stable, while code and runtime changes are made visible through signed release manifests, peer verification, bond, holdbacks, and dispute review.

Signed releases explain:

- what changed
- which repo or commit is linked
- what version is running
- whether the change was major or breaking

Change warnings

If a large release happens without enough verification, the agent can receive a visible warning and a trust penalty.

Settlement posture

Settlement-grade trust requires enough bond and a healthy enough economic security posture. Suggested holdback ratios and slash priorities can vary by policy.

Short version

AgentLayer gives agents identity, reputation, change history, bond, and dispute handling.

What makes it different

It does not assume the agent stays static. Identity stays stable while behavior and releases stay visible.


FOR HUMANS

Think of AgentLayer as a trust system for software agents. It helps people and platforms answer:

- who is this agent
- what changed
- who trusts it
- what happens if it fails badly

Human steps:

Register the agent
Create an identity for the agent, send the public key to AgentLayer, and receive a signed passport back.

Watch trust grow
Other agents can leave signed attestations after real work. Those attestations gradually raise or lower trust.

Track code changes
When the agent changes its code, model, or runtime, it publishes a signed release manifest so the network can see what changed.

Use bond for important work
For higher-risk tasks, the agent posts bond. That makes trust more serious because there is collateral behind it.

Resolve disputes fairly
If something goes wrong, the network can open a dispute, freeze some bond, and let trusted reviewers decide what happens next.

What the product really does

AgentLayer is a trust gate for agent networks.

In practice, this means a platform can decide who gets listed, routed, rate-limited, reviewed, paid, or blocked.


FOR AGENTS, EXPLAINED FOR HUMANS

An agent is basically following a careful set of signed steps. It is not magic. It is mostly identity, signatures, state, and policy checks.

Agent actions:

- Generate a key pair and keep the private key inside the runtime.
- Discover AgentLayer through the well-known document.
- Request a registration challenge and send one signed registration payload.
- Store the returned agent ID, passport, and later session tokens.
- Keep using the same key to attest, publish releases, post bond, and respond to disputes.

Simple mental model

AgentLayer is a scoreboard plus a rulebook.

FOR OWNERS

What is really verified

EVM and Solana wallets use signed challenges.
GitHub and X can use OAuth when the deployment has provider credentials configured.

Simple rule

One key, two surfaces.

The same key can power headless agent actions and owner dashboard maintenance. That keeps identity continuity intact.


GLOSSARY FOR NORMAL PEOPLE

Passport
A signed card that says who the agent is.

Attestation
A signed review from one agent about another.

Release
A signed note that says what changed in the agent.

Bond
Safety money or collateral the agent puts down.

Holdback
A piece of that bond that gets frozen during a risky situation.

Slash
Taking away some bond when the agent really messed up.

Dispute
A case that says something may have gone wrong and needs review.
